Introduction

This assignment will give you an opportunity to create a Graphical User Interface (GUI) and to apply the techniques for designing good software that you have been learning in class.

Part 1 of the assignment requires you to create a CSSE1001/7030 queueing application to mimic the one that is used currently for CSSE1001/7030 pracs.

An example of what this queue should look like is below

Basic GUI Example

Unlike the existing web queue, however, the application you write is to be desktop application accessed rather than a website queue. The queue you produce must be written entirely in python using tkinter. You are not required to make the application work with multiple users over a network.

The GUI in your software is to have the same labels, buttons and text as the existing web application, but the "join queue" buttons will have slightly different functions. When a user presses either the "Request quick help" button or "Request long help" button, your software must display a dialog box which asks the user for their name. Once they have entered their name, your software then needs to join the user to the queue, display their name on the screen, and regularly update the amount of time they have been waiting on the queue.

When a student is added to the queue, they must be added in such a way that no student who has asked a greater number of questions in the past 24 hours is ahead of them. A student must also be able to remove themselves from the queue by pressing the red button (leaving voluntarily) or the green button (have had their question answered).

You are not required to implement days changing - the "Questions Today" section should be the questioned asked since opening the application.

  2. Provision of a game which the user can play while they are waiting to have their questions answered

All additional features must be written in Python. To appraise the suitability of a given additional feature, you are advised to consult with tutors or post a private query on Piazza.

For Part 2 of the Assignment you must also submit a pdf document which describes the additional feature(s) you have implemented, a brief user manual on the how to interact with your additional feature software, and the approach you took to writing the code.

Masters Task

Masters students (i.e. CSSE7030 students) must do an additional task. They must create a menubar which has a “Statistics” tab in it. When this tab is activated it displays the options “Mean”, “Median” and “Mode”. When one of these options is selected it triggers respectively a display of the mean, median and modal time to wait for questions to be answered on both the short and long question queues for the previous day.
